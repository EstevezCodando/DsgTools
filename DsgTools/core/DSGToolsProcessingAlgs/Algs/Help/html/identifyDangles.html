Descrição:

Este algoritmo provê uma interface de modo que o usuário consiga verificar se uma camada respeita um conjunto de regras para o preenchimento correto de atributos ou conjunto de atributos.

Parâmetros:

• Camada de entrada → Camada a ser processada em busca de linhas desconectadas. • input is a boundary layer ( every line must be connected to an element of either the input layer or the filters) → Cria os limites da camada de entrada, quando for usado os filtros opcionais e o parâmetro Geographic Boundary. • Raio de busca → Distância mínima de uma feição a outra (de uma camada que devesse ser atraí-la) para que não seja considerada desconectada (ponta solta). • Camada de filtro do tipo linha → Camadas do tipo linhas a serem adicionadas à regra de checagem de proximidade quando for buscar feições próximas à analisada. • Camada de filtro do tipo polígono → Camadas do tipo polígono a serem adicionadas à regra de checagem de proximidade quando for buscar feições próximas à analisada. • Ignorar pontas soltas em linhas não segmentadas → Indica se o algoritmo deve ignorar extremidades de linha que tocam um segmento de reta sem que haja ponto de interseção. • Geographic Boundary → Será utilizada uma camada do tipo poligono para filtrar e identificar pontas soltas contidas dentro dessa camada. Atenção: Utilização do raio em seu sistema de coordenadas da camada de entrada.

Exemplo de uso:

Modelagem EDGV3.0 na escala de 1:25.000 (EPSG):4674

• Camada de entrada → hid_trecho_drenagem_l • input is a boundary layer → false • Raio de busca → 0,000100 • Camada de filtro do tipo linha → opicional • Camada de filtro do tipo polígono → opicional • Ignorar pontas soltas em linhas não segmentadas → false
